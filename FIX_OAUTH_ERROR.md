# 🔧 Решение ошибки "invalid_request" при авторизации ЮMoney

Если вы получили ошибку **"Этому приложению отказано в доступе. invalid_request"**, это означает проблему с настройками OAuth приложения.

---

## 🔍 Причины ошибки

Ошибка `invalid_request` возникает когда:

1. **Redirect URI не совпадает** - параметр `redirect_uri` в запросе должен **точно** совпадать с тем, что указано в настройках приложения в кабинете ЮMoney
2. **Приложение не настроено** - приложение не существует или не активировано
3. **Неправильные параметры** - отсутствуют или некорректны обязательные параметры запроса

---

## ✅ Решение 1: Проверка настроек приложения (ОБЯЗАТЕЛЬНО)

### Шаг 1: Войдите в кабинет разработчика ЮMoney

1. Перейдите на https://yoomoney.ru/
2. Войдите в свой аккаунт
3. Перейдите в **"Разработчикам"** → **"Приложения"**
   - Или прямой URL: https://yoomoney.ru/myservices/new

### Шаг 2: Найдите ваше приложение

Приложение называется **"Труба VPN"** с идентификатором:
```
5DE3736B6BA71E9420E1B269CD147ACD8AEA6F69AE4FE87E5AD80983E49A2FD0
```

### Шаг 3: Проверьте Redirect URI

В настройках приложения должно быть указано **ОДИН** из следующих Redirect URI:

**Вариант А (рекомендуется для desktop приложений):**
```
urn:ietf:wg:oauth:2.0:oob
```

**Вариант Б:**
```
https://yoomoney.ru
```

**Вариант В:**
```
http://localhost
```

⚠️ **ВАЖНО:** Redirect URI в настройках приложения должен **точно совпадать** с тем, что использует скрипт!

---

## ✅ Решение 2: Изменить Redirect URI в скрипте

Если вы не можете изменить настройки приложения, измените `redirect_uri` в скрипте:

### Способ А: Изменить в файле yoomoney_oauth.py

Откройте `yoomoney_oauth.py` и измените строку:

```python
REDIRECT_URI = os.getenv("YOOMONEY_REDIRECT_URI", "urn:ietf:wg:oauth:2.0:oob")
```

Замените на один из вариантов:

**Для desktop приложений (рекомендуется):**
```python
REDIRECT_URI = "urn:ietf:wg:oauth:2.0:oob"
```

**Если в настройках указано:**
```python
REDIRECT_URI = "https://yoomoney.ru"
```

**Или:**
```python
REDIRECT_URI = "http://localhost"
```

### Способ Б: Через переменную окружения

Установите переменную окружения перед запуском скрипта:

**Windows PowerShell:**
```powershell
$env:YOOMONEY_REDIRECT_URI="urn:ietf:wg:oauth:2.0:oob"
python yoomoney_oauth.py
```

**Windows CMD:**
```cmd
set YOOMONEY_REDIRECT_URI=urn:ietf:wg:oauth:2.0:oob
python yoomoney_oauth.py
```

**Linux/Mac:**
```bash
export YOOMONEY_REDIRECT_URI="urn:ietf:wg:oauth:2.0:oob"
python yoomoney_oauth.py
```

## ✅ Решение 3: Использовать правильный Redirect URI в настройках приложения

### Если используете `urn:ietf:wg:oauth:2.0:oob`:

1. Войдите в кабинет разработчика ЮMoney
2. Найдите приложение "Труба VPN"
3. В настройках приложения в поле **"Redirect URI"** укажите:
   ```
   urn:ietf:wg:oauth:2.0:oob
   ```
4. Сохраните изменения

### После изменения:

1. Подождите несколько минут (настройки могут применяться с задержкой)
2. Повторите попытку авторизации

## 🔄 Полный процесс исправления (пошагово)

### Шаг 1: Проверьте текущий redirect_uri в скрипте

Откройте `yoomoney_oauth.py` и найдите строку:
```python
REDIRECT_URI = os.getenv("YOOMONEY_REDIRECT_URI", "urn:ietf:wg:oauth:2.0:oob")
```

Запомните это значение.

### Шаг 2: Войдите в кабинет разработчика ЮMoney

1. https://yoomoney.ru/
2. Войдите → Разработчикам → Приложения

### Шаг 3: Откройте настройки приложения

Найдите приложение с `client_id`: `5DE3736B6BA71E9420E1B269CD147ACD8AEA6F69AE4FE87E5AD80983E49A2FD0`

### Шаг 4: Проверьте/измените Redirect URI

В настройках приложения в поле **Redirect URI** должно быть указано:

**Если в скрипте:**
- `urn:ietf:wg:oauth:2.0:oob` → в настройках: `urn:ietf:wg:oauth:2.0:oob`
- `https://yoomoney.ru` → в настройках: `https://yoomoney.ru`
- `http://localhost` → в настройках: `http://localhost`

⚠️ **Должно совпадать ТОЧНО, включая протокол и регистр!**

### Шаг 5: Сохраните и повторите

1. Сохраните изменения в кабинете ЮMoney
2. Подождите 1-2 минуты
3. Запустите скрипт заново: `python yoomoney_oauth.py`
4. Попробуйте авторизоваться снова

---

## 🧪 Проверка после исправления

После настройки попробуйте:

```bash
python yoomoney_oauth.py
```

Если все настроено правильно, вы увидите страницу авторизации ЮMoney без ошибки.

---

## 📞 Если ничего не помогло

Если после всех исправлений ошибка сохраняется:

1. **Проверьте, что приложение активно** - в кабинете должно быть написано "Активно" или "Включено"
2. **Убедитесь в правильности client_id и client_secret**
3. **Попробуйте создать новое приложение** в кабинете ЮMoney:
   - Создайте новое приложение
   - Укажите Redirect URI: `urn:ietf:wg:oauth:2.0:oob`
   - Скопируйте новый `client_id` и `client_secret`
   - Обновите их в `yoomoney_oauth.py`

---

## 💡 Рекомендации

**Лучший вариант для desktop приложений:**
- Используйте `urn:ietf:wg:oauth:2.0:oob` - это стандартный вариант для приложений без веб-сервера
- Этот вариант не требует настройки веб-сервера
- Код авторизации будет показан на странице после авторизации

**Альтернатива:**
- Если `urn:ietf:wg:oauth:2.0:oob` не работает, используйте `https://yoomoney.ru` или `http://localhost`
- Убедитесь, что выбранный вариант указан в настройках приложения
