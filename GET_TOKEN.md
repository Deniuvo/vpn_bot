# Инструкция: Как получить API токен доступа ЮMoney

Это руководство поможет вам получить токен доступа (access token) для автоматической проверки платежей через API ЮMoney.

---

## 📋 Что вам понадобится

- Учетная запись ЮMoney
- Python 3.x установлен на компьютере
- Библиотека `yoomoney` и `requests` установлены

---

## 🔧 Шаг 1: Установка зависимостей

Убедитесь, что установлены необходимые библиотеки:

```bash
pip install yoomoney requests
```

Или установите все зависимости из requirements.txt:

```bash
pip install -r requirements.txt
```

---

## 🚀 Шаг 2: Получение токена (автоматический способ)

### 2.1. Запустите скрипт для получения URL авторизации

Откройте терминал/командную строку в папке с проектом и выполните:

```bash
python yoomoney_oauth.py
```

Скрипт покажет вам URL для авторизации. Вы увидите что-то вроде:

```
============================================================
ШАГ 1: Откройте этот URL в браузере:
============================================================
https://yoomoney.ru/oauth/authorize?client_id=5DE3736B6BA71E9420E1B269CD147ACD8AEA6F69AE4FE87E5AD80983E49A2FD0&response_type=code&redirect_uri=https%3A%2F%2Fyoomoney.ru&scope=account-info+operation-history+operation-details+incoming-transfers
============================================================
```

### 2.2. Откройте URL в браузере

Скопируйте весь URL, который показал скрипт, и откройте его в браузере.

Вы увидите страницу авторизации ЮMoney.

⚠️ **Если получили ошибку "invalid_request" или "Этому приложению отказано в доступе":**
- Это означает проблему с настройками Redirect URI
- См. подробное решение в файле **`FIX_OAUTH_ERROR.md`**
- Основная причина: Redirect URI в настройках приложения не совпадает с тем, что в скрипте

### 2.3. Авторизуйтесь и дайте разрешение

1. Войдите в свой аккаунт ЮMoney (если не авторизованы)
2. Прочитайте список запрашиваемых разрешений:
   - `account-info` - информация об аккаунте
   - `operation-history` - история операций
   - `operation-details` - детали операций
   - `incoming-transfers` - входящие переводы
3. Нажмите кнопку **"Разрешить"** или **"Разрешить доступ"**

### 2.4. Скопируйте код авторизации

После разрешения доступа вас перенаправит на страницу:

```
https://yoomoney.ru?code=ВАШ_КОД_АВТОРИЗАЦИИ
```

⚠️ **ВАЖНО:** Скопируйте значение параметра `code` из URL.

**Пример:**
- Если URL: `https://yoomoney.ru?code=abc123xyz456`
- То код: `abc123xyz456`

💡 **Совет:** Можно скопировать весь URL - скрипт сам извлечет код.

### 2.5. Обменяйте код на токен

Вернитесь в терминал и выполните:

```bash
python yoomoney_oauth.py ВАШ_КОД
```

Замените `ВАШ_КОД` на код, который вы скопировали на шаге 2.4.

**Пример:**
```bash
python yoomoney_oauth.py abc123xyz456
```

Или если скопировали весь URL:
```bash
python yoomoney_oauth.py https://yoomoney.ru?code=abc123xyz456
```

### 2.6. Получите токен

Если все прошло успешно, вы увидите:

```
============================================================
✅ Токен успешно получен!
============================================================

Токен доступа:
4100119393589473-abc123def456...

============================================================

Сохраните этот токен!

Для Windows PowerShell:
$env:YMONEY_ACCESS_TOKEN="ваш_токен"

Для Windows CMD:
set YMONEY_ACCESS_TOKEN=ваш_токен

Для Linux/Mac:
export YMONEY_ACCESS_TOKEN="ваш_токен"
============================================================
```

## 💾 Шаг 3: Сохранение токена

После получения токена его нужно сохранить в переменную окружения или в файл конфигурации.

### Вариант А: Переменная окружения (рекомендуется)

#### Для Windows PowerShell:
```powershell
$env:YMONEY_ACCESS_TOKEN="ваш_токен_доступа"
```

Чтобы токен сохранялся после перезапуска PowerShell, добавьте его в профиль:
```powershell
[System.Environment]::SetEnvironmentVariable("YMONEY_ACCESS_TOKEN", "ваш_токен_доступа", "User")
```

#### Для Windows CMD:
```cmd
setx YMONEY_ACCESS_TOKEN "ваш_токен_доступа"
```

После этого перезапустите командную строку.

#### Для Linux/Mac:
```bash
export YMONEY_ACCESS_TOKEN="ваш_токен_доступа"
```

Чтобы сохранить навсегда, добавьте в `~/.bashrc` или `~/.zshrc`:
```bash
echo 'export YMONEY_ACCESS_TOKEN="ваш_токен_доступа"' >> ~/.bashrc
source ~/.bashrc
```

### Вариант Б: Сохранение в bot.py (не рекомендуется для продакшена)

Если вы не хотите использовать переменные окружения, можете временно указать токен прямо в `bot.py`:

```python
YMONEY_ACCESS_TOKEN = os.getenv("YMONEY_ACCESS_TOKEN", "ваш_токен_здесь")
```

⚠️ **ВНИМАНИЕ:** Никогда не коммитьте токен в публичные репозитории!

### Вариант В: Файл .env (для разработки)

Создайте файл `.env` в корне проекта:

```env
YMONEY_ACCESS_TOKEN=ваш_токен_доступа
```

И добавьте `.env` в `.gitignore`, чтобы случайно не закоммитить токен.

## 🔄 Что делать, если что-то пошло не так

### ⚠️ Ошибка: "invalid_request" или "Этому приложению отказано в доступе"

Это самая частая ошибка при авторизации. Она означает, что Redirect URI не совпадает с настройками приложения.

**Быстрое решение:**
1. Откройте кабинет разработчика ЮMoney: https://yoomoney.ru/myservices/new
2. Найдите приложение "Труба VPN" 
3. В настройках приложения проверьте Redirect URI
4. Он должен **точно совпадать** с тем, что использует скрипт (по умолчанию: `urn:ietf:wg:oauth:2.0:oob`)

📖 **Подробное решение:** см. файл **`FIX_OAUTH_ERROR.md`** - там пошаговые инструкции по исправлению.

### Ошибка: "Токен НЕ установлен"

1. Убедитесь, что вы правильно скопировали команду установки токена
2. Перезапустите терминал после установки переменной окружения
3. Проверьте, что используете правильную командную оболочку (PowerShell/CMD/Bash)

### Ошибка: "invalid_grant" или "invalid_client"

1. Убедитесь, что код авторизации не был использован ранее (коды одноразовые)
2. Получите новый код авторизации (повторите шаги 2.1-2.4)
3. Проверьте, что не прошло слишком много времени после получения кода

### Ошибка: "Токен не работает"

1. Токен мог истечь (редко, но возможно)
2. Получите новый токен по этой же инструкции

### Бот не проверяет платежи автоматически

1. Убедитесь, что `USE_YOOMONEY_API="true"` в переменных окружения или коде
2. Проверьте, что токен действительно установлен (шаг 4)
3. Посмотрите логи бота при запуске

## 📝 Альтернативный способ: Получение токена вручную

Если скрипт не работает, вы можете получить токен вручную:

### 1. Откройте URL авторизации:

```
https://yoomoney.ru/oauth/authorize?client_id=5DE3736B6BA71E9420E1B269CD147ACD8AEA6F69AE4FE87E5AD80983E49A2FD0&response_type=code&redirect_uri=https://yoomoney.ru&scope=account-info%20operation-history%20operation-details%20incoming-transfers
```

### 2. Авторизуйтесь и получите код (как в шаге 2.3-2.4)

### 3. Используйте Python для обмена кода на токен:

```python
from yoomoney_oauth import exchange_code_for_token

code = "ВАШ_КОД_ИЗ_URL"
token = exchange_code_for_token(code)
print(f"Токен: {token}")
```

## 📚 Дополнительная информация

- **Срок действия токена:** Токен действует длительное время (обычно несколько лет), но может быть отозван вами в настройках приложения ЮMoney
- **Безопасность:** Храните токен в секрете, не публикуйте его в открытых репозиториях
- **Ограничения API:** API ЮMoney имеет лимиты на количество запросов, но для проверки платежей этого обычно достаточно

## ❓ Частые вопросы

**Q: Нужно ли получать токен каждый раз при запуске бота?**  
A: Нет, токен получается один раз и действует длительное время.

**Q: Могу ли я использовать один токен для нескольких ботов?**  
A: Да, один токен можно использовать для нескольких приложений, если они работают с одним кошельком.

**Q: Что делать, если токен перестал работать?**  
A: Получите новый токен по этой же инструкции.

**Q: Можно ли работать без токена?**  
A: Да, бот будет работать в режиме ручной проверки платежей через скриншоты.
