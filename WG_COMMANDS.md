# 📋 Команды для изменения конфигурации WireGuard

## 🔧 Команды, которые изменяют конфиг WireGuard

### 1. **Добавление peer на сервер** (временное изменение)

**Команда:**
```bash
wg set wg0 peer <PUBLIC_KEY> allowed-ips <IP_ADDRESS>/32
```

**Что делает:**
- Добавляет peer на работающий WireGuard сервер
- ⚠️ **Временное изменение** - после перезапуска WireGuard исчезнет
- Используется ботом автоматически при генерации конфига

**Пример:**
```bash
wg set wg0 peer AbCdEfGhIjKlMnOpQrStUvWxYz1234567890 allowed-ips 10.0.0.5/32
```

**Где используется:**
- `wireguard.py` → функция `add_peer_to_server()`
- Вызывается автоматически при оплате и генерации конфига

---

### 2. **Удаление peer с сервера** (временное изменение)

**Команда:**
```bash
wg set wg0 peer <PUBLIC_KEY> remove
```

**Что делает:**
- Удаляет peer с работающего WireGuard сервера
- ⚠️ **Временное изменение**

**Пример:**
```bash
wg set wg0 peer AbCdEfGhIjKlMnOpQrStUvWxYz1234567890 remove
```

**Где используется:**
- `wireguard.py` → функция `remove_peer_from_server()`

---

### 3. **Изменение конфига сервера** (постоянное изменение)

**Команда:**
```bash
# 1. Редактируем файл
nano /etc/wireguard/wg0.conf

# 2. Применяем изменения
wg-quick down wg0
wg-quick up wg0
```

**Что делает:**
- Редактирует файл `/etc/wireguard/wg0.conf`
- Применяет изменения через перезапуск
- ✅ **Постоянное изменение** - сохраняется после перезагрузки

**Пример изменения:**
```ini
# Добавляем peer вручную в конфиг
[Peer]
PublicKey = AbCdEfGhIjKlMnOpQrStUvWxYz1234567890
AllowedIPs = 10.0.0.5/32
```

**Где используется:**
- При ручной настройке WireGuard
- В скриптах установки (`setup_wireguard.sh`)

---

### 4. **Создание конфига клиента** (создание файла)

**Команда:**
```python
wg_manager.create_config(user_id, private_key, ip_address)
```

**Что делает:**
- Создает файл конфига для клиента в `configs/user_id.conf`
- ✅ **Постоянное изменение** - файл сохраняется

**Где используется:**
- `bot.py` → функция `process_payment_and_generate_config()`
- Вызывается автоматически при оплате

---

### 5. **Обновление URL конфига в базе данных**

**Команда:**
```python
db.update_user_config(user_id, config_url)
```

**Что делает:**
- Обновляет только URL конфига в базе данных SQLite
- Не меняет сам файл конфига

**Где используется:**
- `bot.py` → после генерации конфига

---

## 📊 Сравнение команд

| Команда | Тип изменения | Сохраняется после перезапуска? | Где используется |
|---------|--------------|-------------------------------|------------------|
| `wg set wg0 peer ...` | Временное | ❌ Нет | Автоматически ботом |
| `wg-quick down/up wg0` | Постоянное | ✅ Да | При изменении конфига сервера |
| `create_config()` | Постоянное | ✅ Да | Автоматически ботом |
| Редактирование `/etc/wireguard/wg0.conf` | Постоянное | ✅ Да | Вручную |

---

## 🎯 Основная проблема

**Почему peer исчезает после перезапуска?**

Команда `wg set` делает **временное** изменение. Для постоянного сохранения нужно:

1. **Вариант 1:** Добавить peer в `/etc/wireguard/wg0.conf`:
```bash
nano /etc/wireguard/wg0.conf
# Добавьте секцию [Peer]
wg-quick down wg0
wg-quick up wg0
```

2. **Вариант 2:** Использовать скрипт, который сохраняет в конфиг:
```bash
./fix_peer_manual.sh <PUBLIC_KEY> <IP>
# Скрипт спросит, сохранить ли в конфиг
```

---

## 💡 Рекомендации

1. **После добавления peer через `wg set`** - сохраните в конфиг для постоянства
2. **После изменения `/etc/wireguard/wg0.conf`** - всегда перезапускайте через `wg-quick`
3. **Проверяйте изменения:**
   ```bash
   wg show wg0  # Показать текущее состояние
   ```

## 🔍 Полезные команды для проверки

```bash
# Показать все peers
wg show wg0

# Показать только публичные ключи peers
wg show wg0 peers

# Показать статистику передачи данных
wg show wg0 transfer

# Проверить конфиг файл
cat /etc/wireguard/wg0.conf
```
