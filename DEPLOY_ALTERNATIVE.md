# 🔧 Решение проблемы подключения к серверу

## ❌ Ошибка: "Connection timed out"

Эта ошибка означает, что сервер недоступен или SSH порт закрыт.

---

## 🔍 Возможные причины:

1. **Неправильный IP адрес**
2. **SSH порт не 22** (другой порт)
3. **Файрвол блокирует подключение**
4. **Сервер не доступен из вашей сети**

---

## ✅ Решения:

### Вариант 1: Проверка доступности сервера

```bash
# Проверка доступности сервера
ping 90.156.169.27

# Проверка SSH порта
telnet 90.156.169.27 22
```

Если ping не проходит, сервер недоступен или IP неправильный.

---

### Вариант 2: Другой SSH порт

Если SSH работает на другом порту (например, 2222):

```bash
scp -r -P 2222 vpn_bot root@90.156.169.27:/root/
```

---

### Вариант 3: Использование WinSCP (Windows GUI)

1. Скачайте WinSCP: https://winscp.net/
2. Установите и запустите
3. Создайте новое подключение:
   - Хост: `90.156.169.27`
   - Порт: `22` (или другой, если указан)
   - Пользователь: `root`
   - Пароль: ваш пароль
4. Подключитесь и скопируйте папку `vpn_bot` через интерфейс

---

### Вариант 4: Использование Git (если есть доступ к GitHub/GitLab)

Если у вас есть Git репозиторий:

**На сервере:**
```bash
cd /root
git clone https://github.com/ваш-username/vpn_bot.git
cd vpn_bot
```

**Или через приватный репозиторий:**
```bash
git clone git@github.com:ваш-username/vpn_bot.git
```

---

### Вариант 5: Ручное создание файлов на сервере

1. **Подключитесь к серверу** через веб-консоль панели управления (если доступна)

2. **Создайте структуру файлов вручную:**

```bash
# На сервере
mkdir -p /root/vpn_bot
cd /root/vpn_bot

# Создайте файлы через nano/vim или через веб-интерфейс
```

3. **Или используйте FTP/SFTP клиент**

---

### Вариант 6: Архив и загрузка через веб-интерфейс

1. **Создайте архив на локальном компьютере:**

```bash
# В PowerShell
cd Desktop
Compress-Archive -Path vpn_bot -DestinationPath vpn_bot.zip
```

2. **Загрузите через веб-интерфейс хостинга** (если доступен)

3. **Распакуйте на сервере:**

```bash
cd /root
unzip vpn_bot.zip
```

---

### Вариант 7: Через VPN или туннель

Если сервер доступен только через VPN:

1. Подключитесь к VPN сервера
2. Затем используйте SCP как обычно

---

## 🔧 Проверка настроек сервера

Если у вас есть доступ к серверу другим способом, проверьте:

### 1. Проверка SSH сервиса

```bash
# На сервере
sudo systemctl status ssh
# или
sudo systemctl status sshd

# Если не запущен
sudo systemctl start ssh
sudo systemctl enable ssh
```

### 2. Проверка файрвола

```bash
# UFW
sudo ufw allow 22/tcp
sudo ufw status

# iptables
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
```

### 3. Проверка SSH конфигурации

```bash
# Проверка, слушает ли SSH
sudo netstat -tlnp | grep :22
# или
sudo ss -tlnp | grep :22
```

---

## 🎯 Рекомендуемый порядок действий:

1. **Проверьте IP адрес** - убедитесь, что он правильный
2. **Попробуйте WinSCP** - более удобно для Windows
3. **Используйте веб-консоль** хостинга для ручного копирования
4. **Или используйте Git** - если есть репозиторий

---

## 💡 Быстрое решение для Windows:

**Используйте WinSCP или FileZilla:**
- WinSCP: https://winscp.net/ (рекомендуется)
- FileZilla: https://filezilla-project.org/

Оба поддерживают SFTP и более удобны чем командная строка.

---

## ⚠️ Важно:

- Убедитесь, что у вас есть доступ к серверу
- Проверьте, что SSH порт открыт
- Используйте правильный IP адрес и порт
- Если сервер в другой сети, может потребоваться VPN
