# 📦 Альтернативные способы копирования файлов на сервер

## ✅ Сервер доступен, но SSH порт 22 закрыт

Сервер отвечает на ping, но SSH недоступен. Вот несколько способов:

---

## 🎯 Вариант 1: WinSCP (РЕКОМЕНДУЕТСЯ для Windows)

### Шаг 1: Скачайте WinSCP
- Сайт: https://winscp.net/
- Скачайте установщик для Windows

### Шаг 2: Установите и запустите

### Шаг 3: Настройте подключение
- **Протокол:** SFTP или SCP
- **Имя хоста:** `90.156.169.27`
- **Порт:** `22` (или другой, если указан в панели управления)
- **Имя пользователя:** `root`
- **Пароль:** ваш пароль root

### Шаг 4: Подключитесь и скопируйте
1. Нажмите "Войти"
2. После подключения перетащите папку `vpn_bot` из левой панели (локальный компьютер) в правую (сервер) в папку `/root/`

**Преимущества:** Визуальный интерфейс, поддержка разных портов

---

## 🎯 Вариант 2: Создать архив и загрузить через веб-панель

### Шаг 1: Создайте архив на Windows

**PowerShell:**
```powershell
cd Desktop
Compress-Archive -Path vpn_bot -DestinationPath vpn_bot.zip
```

Или вручную через Проводник:
1. Правый клик на папку `vpn_bot`
2. "Отправить" → "Сжатая ZIP-папка"

### Шаг 2: Загрузите через веб-интерфейс хостинга
- Войдите в панель управления сервером (если доступна)
- Найдите файловый менеджер
- Загрузите `vpn_bot.zip`

### Шаг 3: Распакуйте на сервере
- Через файловый менеджер распакуйте архив
- Или через веб-консоль выполните:
```bash
cd /root
unzip vpn_bot.zip
```

---

## 🎯 Вариант 3: Использовать Git (если есть репозиторий)

### Если у вас есть GitHub/GitLab:

**На сервере** (через веб-консоль или другой доступ):
```bash
cd /root
git clone https://github.com/ваш-username/vpn_bot.git
# или
git clone git@github.com:ваш-username/vpn_bot.git
```

---

## 🎯 Вариант 4: Проверить SSH порт

Возможно, SSH работает на другом порту. Попробуйте:

### Через WinSCP:
- Используйте порт `2222`, `22222` или другой (уточните у хостинг-провайдера)

### Через командную строку:
```bash
scp -r -P 2222 vpn_bot root@90.156.169.27:/root/
```

---

## 🎯 Вариант 5: Открыть SSH порт на сервере

Если у вас есть доступ к серверу другим способом (веб-консоль):

```bash
# Проверка SSH сервиса
sudo systemctl status ssh
sudo systemctl start ssh
sudo systemctl enable ssh

# Открыть порт в файрволе (UFW)
sudo ufw allow 22/tcp
sudo ufw reload

# Или для iptables
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
```

---

## 🎯 Вариант 6: Ручное создание через веб-консоль

1. **Откройте веб-консоль** сервера (обычно доступна в панели управления)

2. **Создайте файлы:**

```bash
mkdir -p /root/vpn_bot
cd /root/vpn_bot

# Создайте файлы через nano
nano bot.py
# (вставьте содержимое файла)
```

3. **Или используйте файловый менеджер** в панели управления

---

## 📋 Рекомендуемый порядок действий:

### Самый простой способ для Windows:

1. ✅ **Скачайте WinSCP** - https://winscp.net/
2. ✅ **Установите и запустите**
3. ✅ **Подключитесь к серверу** (проверьте порт SSH)
4. ✅ **Скопируйте папку** `vpn_bot` в `/root/`

### Если WinSCP не работает:

1. ✅ **Создайте ZIP архив** папки `vpn_bot`
2. ✅ **Загрузите через веб-панель** хостинга
3. ✅ **Распакуйте на сервере**

---

## ⚠️ Важно:

- Убедитесь, что знаете правильный **SSH порт** (может быть не 22)
- Проверьте настройки файрвола на сервере
- Используйте веб-консоль хостинга, если SSH недоступен

---

## 🔍 Как узнать SSH порт:

- Посмотрите в панели управления хостинга
- Проверьте документацию провайдера
- Обычные альтернативные порты: 2222, 22222, 22022

**Какой у вас хостинг-провайдер? Это поможет дать более точные инструкции.**
