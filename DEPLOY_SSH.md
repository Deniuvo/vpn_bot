# 🚀 Автоматическая установка через SSH

Несколько способов автоматической установки VPN бота на удаленном сервере через SSH.

---

## 📋 Варианты установки

### Вариант 1: Полный автоматический скрипт (рекомендуется)

**Файл:** `deploy_to_server.sh`

Интерактивный скрипт с проверками и подробным выводом.

**Использование:**

```bash
# Сделать исполняемым
chmod +x deploy_to_server.sh

# Запустить
./deploy_to_server.sh
```

Скрипт попросит:
- IP адрес сервера
- Имя пользователя (обычно root)
- Метод аутентификации (пароль или SSH ключ)

**Что делает:**
- ✅ Проверяет подключение к серверу
- ✅ Устанавливает зависимости Python
- ✅ Устанавливает и настраивает WireGuard с защитой SSH
- ✅ Настраивает переменные окружения
- ✅ Показывает итоговые данные

---

### Вариант 2: Простой скрипт (быстрый старт)

**Файл:** `deploy_simple.sh`

Минималистичный скрипт для быстрого запуска.

**Использование:**

```bash
# Сделать исполняемым
chmod +x deploy_simple.sh

# Запустить (укажите user@server)
./deploy_simple.sh root@192.168.1.100
```

**Требования:**
- SSH ключи настроены (без пароля)
- Или используйте `sshpass` для пароля

**С sshpass (если нужен пароль):**

```bash
sshpass -p 'ваш_пароль' ./deploy_simple.sh root@192.168.1.100
```

---

### Вариант 3: Генератор команд (для копирования)

**Файл:** `generate_ssh_commands.sh`

Создает файл со всеми командами для ручного выполнения.

**Использование:**

```bash
# Создать файл с командами
./generate_ssh_commands.sh

# Файл ssh_commands.txt будет создан
# Скопируйте команды из него и вставьте в SSH консоль сервера
```

Этот вариант полезен, если:
- Нет возможности запустить скрипт автоматически
- Нужно выполнить команды вручную
- Хотите видеть, что именно выполняется

## 🔑 Настройка SSH аутентификации

### Способ 1: SSH ключи (рекомендуется)

**Генерация ключа:**

```bash
ssh-keygen -t rsa -b 4096
```

**Копирование ключа на сервер:**

```bash
ssh-copy-id user@server_ip
```

**Или вручную:**

```bash
cat ~/.ssh/id_rsa.pub | ssh user@server_ip "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

### Способ 2: Пароль (менее безопасно)

Для использования пароля установите `sshpass`:

**Ubuntu/Debian:**
```bash
sudo apt install sshpass
```

**Mac:**
```bash
brew install sshpass
```

**Использование:**

```bash
# В скрипте deploy_to_server.sh выберите опцию "1" при запросе метода аутентификации
```

## 🔧 Устранение проблем

### Ошибка подключения

```
❌ Не удалось подключиться к серверу
```

**Решения:**
1. Проверьте IP адрес сервера
2. Проверьте, что SSH сервис запущен: `sudo systemctl status ssh`
3. Проверьте файрвол на сервере
4. Проверьте SSH ключи или пароль

### Ошибка прав доступа

```
Permission denied (publickey)
```

**Решения:**
1. Проверьте SSH ключ: `ssh -v user@server`
2. Убедитесь, что ключ добавлен на сервер
3. Проверьте права на ключ: `chmod 600 ~/.ssh/id_rsa`

### Ошибка установки WireGuard

Если установка WireGuard не удалась:
1. Проверьте доступ к интернету на сервере
2. Проверьте, что `apt update` работает
3. Выполните установку вручную (см. `WIREGUARD_SETUP.md`)

## 📋 Что устанавливается

Автоматические скрипты выполняют:

1. **Установка Python зависимостей:**
   - Создание виртуального окружения
   - Установка пакетов из `requirements.txt`

2. **Установка WireGuard:**
   - Установка WireGuard и инструментов
   - Генерация ключей сервера
   - Создание конфигурации **с защитой SSH**
   - Настройка IP forwarding
   - Открытие портов в файрволе
   - Запуск WireGuard

3. **Настройка переменных окружения:**
   - `SERVER_IP` - публичный IP сервера
   - `SERVER_PUBLIC_KEY` - публичный ключ WireGuard
   - `YMONEY_ACCESS_TOKEN` - токен API ЮMoney
   - `USE_YOOMONEY_API` - включение API

4. **Проверка:**
   - Проверка подключения
   - Проверка установленных переменных
   - Проверка работы WireGuard

## 🚀 После установки

После успешной установки запустите бота:

```bash
# Подключитесь к серверу
ssh user@server_ip

# Перейдите в директорию
cd /root/vpn_bot

# Активируйте окружение
source vpn_env/bin/activate

# Запустите бота
python3 main.py
```

**Или используйте systemd для автозапуска** (см. `DEPLOY.md`).
