# 📦 Копирование файлов на сервер

Перед установкой WireGuard и запуском бота нужно скопировать все файлы на сервер.

---

## 🚀 Быстрый способ (рекомендуется)

### Вариант 1: Автоматический скрипт

```bash
# Сделайте скрипт исполняемым
chmod +x copy_to_server.sh

# Запустите (укажите ваш сервер)
./copy_to_server.sh root@ваш_сервер
```

Скрипт автоматически:
- ✅ Соберет все необходимые файлы
- ✅ Скопирует их на сервер в `/root/vpn_bot`
- ✅ Покажет следующие шаги

---

### Вариант 2: Ручное копирование через SCP

**Если вы на Linux/Mac:**

```bash
# Перейдите в директорию проекта (на вашем компьютере)
cd ~/Desktop/vpn_bot

# Скопируйте всю папку на сервер
scp -r . root@ваш_сервер:/root/vpn_bot
```

**Если вы на Windows:**

Используйте WinSCP или другой SFTP клиент:

1. Скачайте WinSCP: https://winscp.net/
2. Подключитесь к серверу
3. Перетащите папку `vpn_bot` в `/root/`

## ✅ Проверка после копирования

После копирования файлов подключитесь к серверу и проверьте:

```bash
ssh root@ваш_сервер
cd /root/vpn_bot
ls -la
```

Должны быть видны все файлы, включая:
- ✅ `setup_wireguard.sh`
- ✅ `bot.py`
- ✅ `requirements.txt`

---

## 🔧 Если файлы уже на сервере

Если вы уже копировали файлы ранее, но скрипт `setup_wireguard.sh` не найден:

### Проверьте директорию:

```bash
ssh root@ваш_сервер
cd /root/vpn_bot
ls -la setup_wireguard.sh
```

### Если файла нет, скопируйте его отдельно:

```bash
# С вашего компьютера
scp setup_wireguard.sh root@ваш_сервер:/root/vpn_bot/
```

Или скопируйте весь проект заново.

---

## 📝 Следующие шаги после копирования

После успешного копирования файлов:

1. **Установите зависимости:**
   ```bash
   cd /root/vpn_bot
   python3 -m venv vpn_env
   source vpn_env/bin/activate
   pip install -r requirements.txt
   ```

2. **Установите WireGuard:**
   ```bash
   sudo bash setup_wireguard.sh
   ```

3. **Настройте переменные окружения** (см. `SERVER_SETUP_CHECKLIST.md`)

4. **Запустите бота:**
   ```bash
   python3 main.py
   ```

## 🆘 Решение проблем

### Ошибка "Permission denied"

```bash
# Убедитесь, что у вас есть SSH ключ или пароль
ssh root@ваш_сервер
```

### Ошибка "Connection refused"

- Проверьте, что SSH порт открыт
- Проверьте IP адрес сервера
- Проверьте файрвол на сервере

### Ошибка "No such file or directory"

- Убедитесь, что вы в правильной директории на вашем компьютере
- Проверьте путь к файлам
- Используйте `pwd` для проверки текущей директории
