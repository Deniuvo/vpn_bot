# 🔄 Восстановление peers при перезапуске

## ✅ Проблема решена автоматически!

Система теперь **автоматически восстанавливает** всех активных пользователей при старте бота и правильно сохраняет peers в конфиг.

---

## 🛠️ Что было исправлено:

### 1. **Улучшено сохранение peer в конфиг:**

- Используется `sudo` для записи в `/etc/wireguard/wg0.conf`
- Если `sudo` недоступен, пробует записать напрямую (если бот запущен от root)
- Peer сохраняется **постоянно** и не исчезает при перезапуске WireGuard

### 2. **Автоматическое восстановление при старте бота:**

- При запуске бота автоматически восстанавливаются все активные peers из базы данных
- Не нужно вручную добавлять пользователей после перезапуска

### 3. **Скрипт для ручного восстановления:**

- `restore_peers.py` - скрипт для восстановления всех активных peers
- `restore_peers.sh` - обертка со всеми переменными окружения

---

## 📋 Использование:

### Автоматическое восстановление:

**При старте бота** peers восстанавливаются автоматически - ничего делать не нужно!

```bash
# Просто запустите бота как обычно
cd /root/vpn_bot
bash start_bot.sh
```

Бот автоматически:
1. ✅ Восстановит всех активных пользователей из базы данных
2. ✅ Добавит их peers на WireGuard сервер
3. ✅ Сохранит в конфиг для постоянства

---

### Ручное восстановление (если нужно):

Если нужно восстановить peers вручную:

```bash
cd /root/vpn_bot
chmod +x restore_peers.sh

# Обновите значения SERVER_IP и SERVER_PUBLIC_KEY в restore_peers.sh
# (скопируйте из start_bot.sh)

./restore_peers.sh
```

Или напрямую через Python:

```bash
cd /root/vpn_bot
source vpn_env/bin/activate
export SERVER_IP="your_server_ip"
export SERVER_PUBLIC_KEY="your_public_key"
python3 restore_peers.py
```

---

## 🔧 Что происходит:

### При добавлении нового пользователя:

1. Peer добавляется на WireGuard сервер (временно)
2. Peer **сохраняется в `/etc/wireguard/wg0.conf`** (постоянно)
3. Информация сохраняется в базу данных

### При перезапуске WireGuard:

1. Peer **загружается из конфига** автоматически
2. VPN продолжает работать без ручного вмешательства

### При перезапуске бота:

1. Бот **автоматически восстанавливает** всех активных peers
2. Проверяет базу данных на активные подписки
3. Добавляет peers на WireGuard сервер
4. Сохраняет в конфиг (если еще не сохранены)

---

## ✅ Результат:

- ✅ **Peers сохраняются в конфиг** - не исчезают при перезапуске WireGuard
- ✅ **Автоматическое восстановление** - не нужно вручную добавлять после перезапуска
- ✅ **Полная защита** - все peers правильно сохраняются и восстанавливаются

## 🔍 Проверка работы:

### 1. Проверьте сохранение в конфиг:

```bash
# Проверьте, что peer сохранен в конфиг
cat /etc/wireguard/wg0.conf | grep -A 2 "\[Peer\]"

# Должны увидеть секции [Peer] для каждого пользователя
```

### 2. Проверьте логи бота:

```bash
tail -f /root/vpn_bot/vpn.log | grep "Восстановлено"
```

Должна быть строка типа:
```
✅ Восстановлено 5 активных peers при старте
```

### 3. Проверьте активные peers:

```bash
wg show wg0 peers
```

Должны быть видны все активные публичные ключи.

---

## 📝 Важно:

### Если peers все еще не сохраняются:

1. **Проверьте права доступа:**
   ```bash
   ls -la /etc/wireguard/wg0.conf
   # Должны быть права -rw------- или -rw-r--r-- и владелец root
   ```

2. **Проверьте sudo:**
   ```bash
   sudo -l
   # Убедитесь, что текущий пользователь может использовать sudo
   ```

3. **Проверьте логи:**
   ```bash
   tail -f /root/vpn_bot/vpn.log | grep "Peer сохранен"
   ```

4. **Ручное сохранение (если нужно):**
   ```bash
   # Добавьте peer вручную в конфиг
   sudo nano /etc/wireguard/wg0.conf
   
   # Добавьте в конец:
   [Peer]
   PublicKey = <публичный_ключ_пользователя>
   AllowedIPs = <ip_адрес>/32
   
   # Перезапустите WireGuard
   sudo wg-quick down wg0
   sudo wg-quick up wg0
   ```
